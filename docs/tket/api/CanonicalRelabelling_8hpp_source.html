21 // PERMUTATION HASH EXPLANATION:
22 // Certain permutations on the vertices [0,1,2,3,4,5] are represented by an
23 // unsigned value, the "permutation hash". In fact, ANY permutation on a list of
24 // ANY 6 distinct objects can be reduced to one of these by a suitable vertex
25 // relabelling, which is what this class is for.
26 //
27 // Note that not every permutation on [0,1,2,3,4,5] corresponds to a permutation
28 // hash (in fact, very few do); most of them still need relabelling, just as for
29 // arbitrary labels.
30 //
31 // The permutation hashes are done by taking a partition of 6, with parts in
32 // decreasing order, e.g. 6 = 3+2+1 = 4+2 = 3+3 = 2+2+1+1, etc. We remove all
33 // the 1 entries, and stick the digits together into a single decimal:
34 //
35 // 3+2+1 -> 32, 4+2 -> 42, 2+2+1+1 -> 22
36 //
37 // Each digit represents the length of a slice of the 6 elements 012345:
38 //
39 // 32 -> (012)(34)(5), 42 -> (0123)(45), 22 -> (01)(23)(4)(5).
40 //
41 // The notation (abcd) represents a cyclic shift on the elements a,b,c,d.
42 // Thus a -> b -> c -> d -> a.
43 //
44 // EVERY permutation on 6 ARBITRARY distinct objects is equivalent to one of
45 // these, after suitable vertex relabelling. This follows because permutations
46 // can be decomposed into disjoint cycles.
47 //
