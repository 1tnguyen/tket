89  // NOTE: the reason this is all a bit more complicated (and so, the word
90  // "trivial" is a bit unfair) is that we have to allow empty vertices.
91  // With full vertices (every vertex having a token), we can find cycles just
92  // by starting anywhere and going forwards until we hit the start again.
93  // But if some vertices can be empty, we may not be able to go forward
94  // once we hit an empty vertex, so we then have to go backwards also
95  // until we cannot anymore, and finally link the empty end with the nonempty
96  // start vertex to make a cycle.
97  // However, it's really just the same algorithm as the full tokens case.
