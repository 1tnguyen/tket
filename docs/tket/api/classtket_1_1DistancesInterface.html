void tket::DistancesInterface::register_shortest_path ( const std::vector< size_t > &  path)
virtual

If you KNOW a path from v1 to v2 which is shortest, then extra information about distances can be deduced from subpaths (each subpath must also be a shortest path: otherwise, the whole path would not be of minimum length).

Does nothing unless overridden.

Parameters
pathA sequence [v0,v1, v2, ..., vn] of vertices, KNOWN to be a shortest path from v0 to vn. The caller must not call this without being SURE that it really is a shortest path, or incorrect results may occur.
